# 승인 필요없는 유형(report/guide/reference)도 CLI로는 active 승격 불가

- Issue ID: `issue-20260724T122239Z-4b684443`
- Recorded at: 2026-07-24T12:22:39.706477+00:00
- Reported by: claude-code
- Reported from: agent
- Area: agent_rules
- Severity: medium
- Release observed: v1-6ca03e60176f
- Status: open

## Summary

list-curation-candidates의 14건 중 report/guide/reference/guide 유형 12건을 '승인 없이 처리 가능'으로 안내했으나, 실제 promote-curation-candidate CLI 경로는 항상 non-automated 모드로 동작해 (1) config.yaml에 approval.knowledge_owner 미설정, (2) 해당 Draft들의 extensions.curation에 review_decision.review_id가 없고 review-curation-candidate의 approve 액션도 이를 생성하지 않아, 승격이 구조적으로 불가능함을 확인함.

## Expected result

OPERATING_RULES.md RB-CUR-006이 설명하는 'policy/guide/decision/spec/reference/report 자동 active 승격' 경로가 CLI를 통해 실행 가능해야 한다.

## Actual result

core/candidates.py의 promote_curation_candidate(automated=True) 경로는 CLI(circled-wiki.py promote-curation-candidate)에서 도달 불가능하며(service.py가 항상 automated=False로 호출), review-curation-candidate의 approve 액션도 review_decision.review_id를 생성하지 않아 모든 유형이 knowledge_owner 미설정 상태에서 승격 자체가 차단됨.

## Impact

지식 정제 파이프라인에서 Owner 승인 없이는 어떤 Draft도 active로 전환할 수 없어, report/guide/reference 유형에 대해 안내된 '자동 승격' 기대가 실제로 충족되지 않음.

## Reproduction or context

python3 .circled-wiki/bin/circled-wiki.py list-curation-candidates 로 pending 후보 확인 후 review-curation-candidate --action approve, promote-curation-candidate 순서로 report 유형 Draft 승격 시도.

## Improvement hint

automated 자동 승격 경로를 노출하는 CLI/MCP 동작을 추가하거나, RB-CUR-006 문서를 현재 CLI 동작(모든 유형이 knowledge_owner + review_decision 필요)에 맞게 정정.
