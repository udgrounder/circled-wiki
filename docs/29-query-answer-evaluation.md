# Circled Wiki 질의 응답 평가 기준

## 목적

이 기준은 Agent가 Circled Wiki의 공식 지식을 이용해 사용자 질문에 답할 때, 검색 성공률만이 아니라 근거·권한·신선도와
불확실성 처리를 평가하기 위한 최소 계약이다. Draft 후보와 Evidence 원문은 검토 인터페이스에서만 다루며, 일반 질의의
공식 절차 근거는 `active` Bundle이어야 한다.

## 판정 항목

| 항목 | 통과 기준 | 실패 처리 |
| --- | --- | --- |
| 근거 | 핵심 주장마다 사용한 Active Bundle ID 또는 Evidence ID를 함께 제시한다. | 근거가 없으면 답을 추정하지 않고 추가 Evidence 또는 검토가 필요하다고 답한다. |
| 상태·권한 | Draft·archived·restricted 문서를 공식 답변의 근거로 사용하지 않는다. | 후보는 검토 대기라고만 설명하고 사용자 실행 지침으로 제시하지 않는다. |
| 신선도 | Runbook의 검토 기한·변경 조건을 확인하고 만료/검토요청 상태를 경고한다. | 최신성이 확인되지 않으면 시점 제한을 명시한다. |
| 범위 | Bundle의 적용 대상·제약을 답변에 반영한다. | 다른 조직·채널·권한으로 일반화하지 않는다. |
| 출처 부족 | 검색 결과가 없거나 Evidence가 부족하면 사실처럼 답하지 않는다. | `needs_review`로 분류하고 필요한 정보·수집 경로를 안내한다. |
| 실행 | 실행 요청은 Active Runbook을 찾은 경우에만 Workflow 경로로 전환한다. | Runbook이 없거나 모호하면 사람 확인을 요청한다. |

## Golden Query 최소 세트

1. SNS 마케팅 시작 방법 — Active Guide/Runbook의 절차와 출처를 인용한다.
2. 근거가 없는 질문 — 무근거 답변을 만들지 않고 Evidence 수집 또는 검토를 요청한다.
3. Draft만 존재하는 주제 — Draft를 사용자 실행 지침으로 노출하지 않는다.
4. 만료 또는 `review_requested` Runbook — 실행 전 검토 경고를 낸다.
5. restricted Evidence 관련 질문 — 접근 제한을 우회하지 않는다.

## 자동화 기준

회귀 테스트는 최소한 검색 결과의 상태 필터, source ID 반환, 무근거 경로, Draft 제외를 검증한다. 실제 Agent 응답 평가는
위 표의 여섯 항목을 채점 가능한 fixture로 변환한 뒤 별도 model/provider 평가 환경에서 실행한다. 이 문서는 모델 호출이나
사용자 승인을 대체하지 않는다.
