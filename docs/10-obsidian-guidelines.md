# Obsidian 사용 가이드

## 1. 목적

Obsidian은 사람이 지식을 읽고 검토하고 보완하는 UI다. Source of Truth는 Git이지만, 사람 사용성은 Obsidian이 담당한다.

권장 Vault 경로는 프로젝트 루트 아래 `knowledge/`다.

## 2. 원칙

- Markdown 친화적 구조를 유지한다.
- 링크 기반 문서 탐색을 적극 사용한다.
- 공식 Bundle은 템플릿 기반으로 작성한다.
- Evidence 원문은 직접 수정하지 않는 것을 원칙으로 한다.
- `index.md`는 폴더 안내와 탐색 시작점으로 사용할 수 있다.
- `log.md`는 변경 이력 기록 문서로 사용할 수 있다.
- Obsidian은 저장소 루트가 아니라 `knowledge/` 폴더를 기준으로 연다.
- 문서와 노트에서는 절대 경로 대신 Vault 내부 상대 경로를 사용한다.
- `.raw/`는 처리 중 임시 작업 공간이므로 사람 편집은 최소화한다.
- 처리 성공 후 원본은 `evidence/`에 보존되고 `.raw/` 항목은 삭제된다.

## 3. 권장 사용 방식

- 도메인별 폴더 브라우징
- Bundle 상호 링크 활용
- 변경 전후 Git diff 검토
- Validator 통과 후 Agent 자동 Commit 결과 검토

## 4. 주의사항

- Obsidian 플러그인이 파일 구조를 임의로 바꾸지 않게 주의한다.
- 링크명 변경 시 Bundle ID와 의미 충돌이 없는지 확인한다.
- 파일 이동 시 링크 자동 갱신과 경로 정체성 변경 영향을 함께 검토한다.
- `index.md`, `log.md`를 일반 개념 문서 이름으로 재사용하지 않는다.
- 특정 컴퓨터의 홈 경로나 마운트 경로를 문서에 남기지 않는다.
